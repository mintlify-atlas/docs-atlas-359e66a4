# Welcome to 256t.org

256t.org is a public specification for content-addressable storage that uses fixed-length, 94-character content identifiers (CIDs). This simple standard provides a predictable format for identifying and retrieving content based on its cryptographic hash and length.

## Why 256t.org?

Content-addressable storage solves fundamental problems in digital content distribution:

- **Eliminate broken links** — Content is identified by what it is, not where it is
- **Guarantee integrity** — CIDs are cryptographically derived from content
- **Enable indefinite caching** — Immutable content can be cached forever
- **Simplify distribution** — Any server can host content matching the CID specification

## How It Works

Every 94-character path uniquely identifies content:

```
https://256t.org/<8-char-length><86-char-hash-or-content>
```

The CID consists of two parts:

1. **8 characters** — Base64URL-encoded length (supports up to 256 terabytes)
2. **Up to 86 characters** — SHA-512 hash (for content > 64 bytes) OR the content itself (for content ≤ 64 bytes)

<Note>
For content 64 bytes or smaller, the base64url-encoded content is shorter than or equal to the hash, so it's stored inline instead.
</Note>

## Quick Example

Here's how to compute a CID in JavaScript:

```javascript
import { computeCID } from '256t-cid';

const content = "Hello, World!";
const cid = computeCID(content);
// Returns: AAAAAAAADUhlbGxvLCBXb3JsZCE (13 bytes inline)
```
